# Split & Merge — что это такое?

> **TL;DR:** гибридный конвейер для редактуры научных текстов по ГОСТ/журнальным стандартам и автоматической верификации источников. Главная идея — не доверять LLM без программной проверки.

---

## Что это

**Split & Merge** — это инструмент для двух задач:

1. **Редактура `.docx`** — приведение научных статей к требованиям ГОСТ и журнальным стандартам: поля страницы, шрифты, транслитерация по ГОСТ 7.79–2000, структура аннотации, библиография по образцу. Работает через `python-docx` — это в ~1500× быстрее ручной правки и принципиально надёжнее, чем «попросить LLM поправить стиль».

2. **Каскад веб-фактчекинга** — автоматическая проверка существования DOI, URL и цитат. Восемь уровней, от самого быстрого и бесплатного к самому мощному и платному.

---

## Зачем это нужно

LLM при редактуре склонна **придумывать** источники — заменять авторов на «более подходящих», добавлять несуществующие DOI, генерировать правдоподобные, но ложные библиографические данные. На тестовом прогоне LLM + researcher-web фальсифицировали 9 из 27 источников. Для документа, который пойдёт в журнал, это катастрофа.

Решение Split & Merge: **любой сгенерированный факт перед записью в `.docx` верифицируется автоматически.** Это называется *Pre- и Post-generation gates*.

---

## Как устроено

**5 ролей:** оркестратор (человек или LLM), рецензент, текстовик, технический исполнитель, факт-чекер.

**10 этапов конвейера редактуры:** критический разбор → декомпозиция → сбор фактов → генерация → повторная верификация → рецензирование → сборка → механическая правка → программная вычитка → финальная проверка.

**Каскад фактчекинга — 8 уровней.** Каждый следующий вызывается только если предыдущий не справился:

| Ур. | Инструмент | Что делает | Стоимость |
|:---:|---|---|---|
| 0.5 | **OpenAlex API** | Мгновенная проверка DOI | Бесплатно (CC0) |
| 1   | researcher-web | Поиск фактов | Бесплатно |
| 2   | Crawl4AI | Базовый HTTP-парсинг | Бесплатно |
| 2.5 | Playwright MCP | Интерактивный браузер (логин, формы) | Бесплатно |
| 3   | Scrapling | Обход Cloudflare | Бесплатно |
| 4   | FireCrawl | Гарантированный markdown | **Платный** — только как резерв |
| 5   | Человек | Ручная верификация | — |
| 6   | Gemini DeepSearch | Независимая аналитика | Опционально |

**Принцип:** на ~95% проверок хватает бесплатных уровней. Платный FireCrawl включается, только когда всё остальное провалилось.

---

## Что в системе особенного

- **Zero Trust к LLM.** Двойная верификация фактов: до генерации (сбор сырья из источников) и после (сверка готовых блоков с сырьём). Никакой сгенерированный текст не попадает в `.docx` без программного подтверждения.
- **Максимум бесплатного.** Базовый стек (OpenAlex + Crawl4AI + Scrapling + Playwright) — open-source, без API-ключей. FireCrawl — только в крайнем случае.
- **Скорость без потери надёжности.** Один Python-скрипт выполняет типовую задачу форматирования за ~0.01 секунды. *«90% результата за 30 минут лучше, чем 100% за 3 часа».*
- **Полная воспроизводимость.** Все скрипты детерминированы, формат входа (`targets.json`) и выхода (текстовые файлы с шапкой `URL/FACT/EXPECT/STATUS`) стандартизован. Любой прогон воспроизводим и проверяем.
- **Живая документация в три слоя.** Витрина для всех (`README.md`), карта для разработчика (`analysis.md`), формальная спецификация для подключения LLM (`pipeline/docs/architecture.md`), cheatsheet с техконстантами (`pipeline/docs/knowledge.md`).
- **Задокументированные грабли.** История провала v2 (фальсификация источников) → v3 (явные запреты в промпте) описана в `architecture.md`. Это снимает вопрос *«почему именно так?»* у любого нового участника.

## Результаты

Тестовый прогон на 3 DOI из авторитетных журналов (Nature Medicine, BMJ, Science):

| DOI | Статья | Статус |
|---|---|---|
| 10.1038/s41591-018-0300-7 | Topol — *Nature Medicine* 2019 | ✅ CONFIRMED |
| 10.1136/bmj.n71 | PRISMA 2020 — *BMJ* 2021 | ✅ CONFIRMED |
| 10.1126/science.aax2342 | Obermeyer — *Science* 2019 | ✅ CONFIRMED |

**3 из 3 подтверждены** на бесплатных уровнях (OpenAlex + Crawl4AI). FireCrawl не понадобился. Пойманы опечатки в исходных DOI — это решается на первом уровне каскада.

## Технический стек

**Язык:** Python 3.11+
**Библиотеки:** `python-docx`, `crawl4ai`, `scrapling[all]`
**API:** OpenAlex REST (CC0, без ключа)
**MCP-серверы:** Context7, Playwright, Fetch, Filesystem, E2B, Exa
**CI:** GitHub Actions + ruff
**Платформы:** Windows (основная), Linux/Mac (поддерживается)
